# Database

## Engine

Supabase PostgreSQL (managed). No MongoDB.

## Connection

The frontend connects to Supabase via the JS client:

```javascript
import { createClient } from "@supabase/supabase-js";

const supabase = createClient(
  process.env.REACT_APP_SUPABASE_URL,
  process.env.REACT_APP_SUPABASE_ANON_KEY
);
```

No backend server needed — the client talks directly to Supabase's REST API.

## Tables

See [[Supabase Setup]] for full schema.

### `gallery`
- Images metadata (title, storage_path, content_type, size)
- Soft-delete via `is_deleted` boolean
- Indexed on `created_at` DESC

### `reviews`
- Client testimonials (author_name, location, rating, text)
- Approval system via `approved` boolean
- Public submissions default to `approved = false`
- Indexed on `created_at` DESC

### `messages`
- Contact form submissions (name, email, phone, message)
- Read status via `read` boolean
- Indexed on `created_at` DESC

## Row Level Security

All tables have RLS enabled. See [[Supabase Setup]] for policies.

## Migration from MongoDB

The old MongoDB collections had the same structure:
- `db.users` → Supabase Auth (no custom users table needed)
- `db.gallery` → `gallery` table
- `db.reviews` → `reviews` table
- `db.messages` → `messages` table

Key differences:
- PostgreSQL uses proper `timestamptz` instead of ISO strings
- UUIDs generated by `gen_random_uuid()` instead of Python
- Soft-delete `is_deleted` preserved from MongoDB pattern
- No `_id` ObjectId — just UUID `id` as primary key
